# 🧑‍💻 내 웹앱 프로젝트 — Claude 규칙서

## 🗣️ 기본 소통 방식
- 모든 답변과 주석은 **한국어**로 작성
- 설명할 때는 초보자도 이해할 수 있게 쉽게 풀어서 말하기
- 코드 변경 전 항상 **무엇을, 왜** 바꾸는지 먼저 설명하기

## 🏗️ 코드 작성 원칙
- 코드는 **간결하고 읽기 쉽게** 작성 (영리한 한 줄보다 명확한 세 줄)
- 함수 하나는 **한 가지 역할만** 담당
- 변수명과 함수명은 **영어**로, 뜻을 명확하게 (ex. `getUserData`, `isLoggedIn`)
- 매직 넘버 금지 — 숫자는 상수로 선언 (ex. `const MAX_RETRY = 3`)

## 🧪 안전하게 개발하기
- **기존 파일을 수정하기 전**, 반드시 백업 또는 변경 사항 확인 요청
- DB나 API 관련 코드는 **실행 전 반드시 검토** 요청
- 프로덕션 환경에 영향을 줄 수 있는 작업은 **⚠️ 경고 표시** 후 진행
- 환경변수(API 키, 비밀번호 등)는 절대 코드에 직접 작성 금지 → `.env` 파일 사용

## 📁 파일 및 폴더 구조
- 새 파일 생성 시 어느 폴더에 넣어야 할지 **항상 경로와 함께** 알려주기
- 임시 테스트 파일은 `temp/` 폴더에 모아두기
- 완성된 기능은 `README.md`에 간단히 기록하기

## 🚀 작업 진행 방식
- 큰 기능을 만들 때는 **플랜 먼저** → 승인 후 코딩 시작
- 한 번에 너무 많이 바꾸지 말고, **작은 단위로 나눠서** 진행
- 막히는 부분이 있으면 혼자 해결하려 하지 말고 **대안 2~3가지** 제시하기

## 🙅 하지 말 것
- 물어보지 않은 기능을 임의로 추가하지 말 것
- 기존에 잘 작동하는 코드를 "더 좋게" 하려고 무단으로 수정하지 말 것
- 에러 메시지를 숨기거나 무시하지 말 것 (반드시 원인 파악 후 처리)
